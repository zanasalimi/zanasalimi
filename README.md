### How I Work

- Ship features end-to-end: UI, API, database, deployment, monitoring.
- Architect for the engineer who comes after me, not just for ship day.
- Production-first mindset: auth, error handling, observability, sane logging.
- Design-system thinking on the frontend, contract-first thinking on the API layer.
- Comfortable making trade-offs and explaining them, including the boring ones.
- Treat AI as a real engineering surface, with prompt design, structured outputs, evals, and guardrails, not "fetch + hope."
